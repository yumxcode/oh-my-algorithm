## 对称性方法论

**R/L 比值陷阱**：比值 = R_std / L_std。弱侧 std 极小会把比值撑大 → 假「严重不对称」。弱侧 std 本身过小才是真问题。

### 步骤 1：绝对阈值（弱侧，多为左）

| 关节 | 冻结 | 已激活 |
|------|------|--------|
| hip_pitch L_std | < 8° | > 12° |
| hip_yaw L_std | < 5° | > 8° |
| knee_pitch L_std | < 15° | > 20° |

### 步骤 2：比值（仅弱侧已激活时）

| R/L | 判定 |
|-----|------|
| 1.0~1.5 | 轻度，可接受 |
| 1.5~3.0 | 中度，需改进 |
| > 3.0 | 严重，单腿策略 |

### 步骤 3：优先级

- L=10°，R=30°，R/L=3.0 → 步态质量问题（弱侧在动）
- L=6°，R=27°，R/L=4.47 → 步态缺失问题（弱侧冻结）
- 综合对称分在弱侧冻结时仍可虚高

### 示例：exp_033 vs exp_034

| | 033 | 034 | 判定 |
|---|---|---|---|
| L hip_pitch std | 6.16° 冻结 | 10.96° 激活 | **034 更好** |
| hip_pitch R/L | 4.47 | 2.79 | 034 更好（原因不同） |
| 综合分数 | 0.734 | 0.585 | 033 分数为假对称 |
| 视觉 | 左腿像棍 | 左腿在跑 | **034 更好** |

## F1 关节符号（摘要）

完整规范：`locomotion-lab/shared/robot_conventions/F1_29DOF_joint_sign_convention.md`

| 关节 | L+ | R+ |
|------|----|----|
| hip_pitch | 前摆抬腿 | 后蹬 |
| hip_roll | 内收 | 外展 |
| hip_yaw | 外旋 | 内旋 |

| base_euler | + 含义 |
|------------|--------|
| roll | 身体向右倾 |
| pitch | 前倾（跑步常见） |
| yaw | 身体向左转 |
